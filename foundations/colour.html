GC Design System · v0.1
Foundations

Colour

Two layers. The neutral ramp is the only raw colour data in the system — perceptually scaled in OKLCH, never referenced by components directly. The semantic surfaces sit on top: tokens like --bg, --fg, --border that resolve to a ramp step via light-dark(). Components consume only these.

Neutral ramp

Thirteen steps from neutral-0 (white) to neutral-1000 (black), perceptually even. Defined in OKLCH so each step looks the same distance from the next; defined as raw values so the semantic layer can pick any step it likes.

Semantic surfaces

Six tokens. Each resolves to one neutral step in light mode and a different step in dark mode, via the native light-dark() function. No duplicated [data-mode] blocks — the same token returns the right value automatically.
